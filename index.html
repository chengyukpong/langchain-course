01 What is LangChain?

The composable toolkit for LLM applications.

LangChain is a framework — think of it as a toolbox full of components you wire together. Its core abstraction is the chain: a sequence of steps that process input and produce output.

The LangChain mental model:

Input → Step 1 → Step 2 → Step 3 → Output

A pipeline. Linear. Predictable. You define the sequence, the LLM fills in the content.

Core components

🔗 Chains / LCEL

  • Pipe components with |
  • Sequential data flow
  • Easy to compose and test

🧰 Tools

  • Functions the LLM can call
  • Search, calculators, APIs
  • Defined with @tool decorator

🤖 Agents

  • LLM decides which tools to use
  • ReAct pattern (reason + act)
  • Simple loop: think → act → observe

💾 Memory

  • Conversation history
  • Buffer, summary, entity types
  • Injected into prompts
LangChain — Simple Agent with Tools Python 
from langchain_openai import ChatOpenAI
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain.tools import tool
from langchain_core.prompts import ChatPromptTemplate

# Define tools
@tool
def search_web(query: str) -> str:
    """Search the web for information."""
    return f"Results for: {query}"

@tool
def calculator(expression: str) -> str:
    """Evaluate a math expression."""
    return str(eval(expression))

# Create agent
llm = ChatOpenAI(model="gpt-4o")
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("placeholder", "{chat_history}"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_tool_calling_agent(llm, [search_web, calculator], prompt)
executor = AgentExecutor(agent=agent, tools=[search_web, calculator])

# Run it
result = executor.invoke({"input": "What's the population of France times 2?"})
print(result["output"])

Key insight: LangChain agents use a single loop — the LLM decides what to do, does it, observes the result, and repeats until it has an answer. There's no explicit graph of states. No cycles you define. Just a loop.

02 LangChain Building Blocks

Understanding the primitives you'll compose with.

LCEL — LangChain Expression Language

The | pipe operator is the backbone. It chains components so the output of one becomes the input of the next.

LCEL — A Simple RAG Chain Python 
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

# The chain: retrieve → format → prompt → LLM → parse
chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# One line to invoke the whole pipeline
answer = chain.invoke("What is LangGraph?")

When LangChain shines

  • RAG pipelines — retrieve, format, generate. Classic linear flow.
  • Simple chatbots — prompt + LLM + memory. Straightforward.
  • Data extraction — structured output from unstructured text.
  • Classification — categorize text with structured output.
  • Summarization chains — map-reduce over documents.

Where LangChain struggles

  • Multi-step reasoning — when the agent needs to plan, revise, backtrack
  • Stateful workflows — tracking progress across many steps
  • Human-in-the-loop — pausing for approval mid-execution
  • Parallel branches — doing multiple things simultaneously
  • Cycles and loops — retry with different strategy, self-correction

03 Why LangGraph Exists

The problem that chains couldn't solve.

Imagine you're building an agent that needs to:

  1. Research a topic by searching multiple sources
  2. Decide if the information is sufficient — if not, search again with refined queries
  3. Draft a summary
  4. Review the summary for accuracy — if it fails review, rewrite it
  5. Format and output the final result

This has cycles (retry loops), branching (sufficient vs. not), state (accumulated research), and conditional routing (pass vs. fail review).

With LangChain's agent loop, you'd hack this into a single prompt and hope the LLM figures out the control flow. It works for simple cases, but as complexity grows, you lose:

❌ What you lose

  • Visibility — can't see where you are in the process
  • Control — can't force a specific next step
  • State — no structured way to accumulate data across steps
  • Recovery — can't resume from a failure point
  • Human oversight — can't pause for approval

✅ What LangGraph gives you

  • Explicit graph — you define the states and transitions
  • Persistence — state saved to a database automatically
  • Cycles — native support for loops and retries
  • Branching — conditional edges based on state
  • Human-in-the-loop — interrupt and resume anywhere
Start
→
Research
→
Enough info?
→
Draft
→
Review pass?
→
Output

↑ This is a graph, not a chain. The "enough info?" and "review pass?" nodes can loop back.

04 LangGraph Core Concepts

Think in graphs, not chains.

The 3 primitives

State

A typed dictionary that flows through the graph. Every node reads from and writes to it.

Nodes

Functions that do work. Call an LLM, run a tool, process data. They take state and return state updates.

Edges

Connections between nodes. Can be unconditional (always go here) or conditional (decide based on state).

LangGraph — Research Agent with Retry Logic Python 
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, Annotated
import operator

# 1. Define state — this is your shared memory
class ResearchState(TypedDict):
    topic: str
    research: Annotated[list[str], operator.add]  # accumulates
    draft: str
    review_passed: bool
    iteration: int

# 2. Define nodes — these do the work
def research(state):
    results = search(state["topic"])
    return {"research": [results], "iteration": state["iteration"] + 1}

def draft(state):
    text = llm.invoke(f"Write about {state['topic']} using: {state['research']}")
    return {"draft": text.content}

def review(state):
    passed = check_quality(state["draft"])
    return {"review_passed": passed}

# 3. Define conditional edges — the routing logic
def should_continue_research(state):
    if len(state["research"]) >= 3:
        return "draft"
    return "research"  # loop back!

def should_retry_draft(state):
    if state["review_passed"]:
        return "output"
    if state["iteration"] >= 3:
        return "output"  # give up, use best effort
    return "research"  # loop back with more research

# 4. Build the graph
graph = StateGraph(ResearchState)
graph.add_node("research", research)
graph.add_node("draft", draft)
graph.add_node("review", review)

graph.add_edge(START, "research")
graph.add_conditional_edges("research", should_continue_research)
graph.add_edge("draft", "review")
graph.add_conditional_edges("review", should_retry_draft)

# 5. Compile and run
app = graph.compile()
result = app.invoke({"topic": "AI agents", "research": [], "iteration": 0})

Notice the difference? In LangChain, you'd encode the retry logic into the prompt. In LangGraph, it's explicit code — you can see every branch, every condition, every loop. The graph is the blueprint of your agent's brain.

05 What is Deep Agents?

The agent harness — batteries-included agents for complex tasks.

Deep Agents is an agent harness — a higher-level layer built on top of LangGraph. While LangChain gives you components and LangGraph gives you orchestration, Deep Agents gives you a complete agent system out of the box.

The three-layer stack:

🧰 Deep Agents
Harness — planning, subagents, filesystem
↕
⚙️ LangGraph
Runtime — state, edges, persistence
↕
🔗 LangChain
Framework — models, tools, prompts

What Deep Agents adds on top

📋 Built-in Planning

  • write_todos tool for task decomposition
  • Agents break complex tasks into steps
  • Track progress and adapt plans

📁 Virtual Filesystem

  • ls, read_file, write_file, edit_file
  • Offload context to file storage
  • Pluggable backends (memory, disk, sandbox)

🤖 Subagents

  • Delegate work to specialized subagents
  • Context isolation between agents
  • Supervisor pattern built-in

🧠 Context Engineering

  • Auto-summarization of long conversations
  • Large tool result eviction
  • Long-term memory across threads

The mental model

LangChain = "Here are the tools. Go figure it out." (LLM controls flow)
LangGraph = "Here's the blueprint. Follow it." (You control flow, LLM does the work)
Deep Agents = "Here's a capable assistant with its own desk, filing cabinet, and subordinates." (Pre-built agent system)

Deep Agents — Minimal Agent with Planning and Filesystem Python 
# pip install deepagents langchain-openai
from deepagents import create_deep_agent

# Create an agent with built-in planning, filesystem, and subagents
agent = create_deep_agent(
    model="openai:gpt-4o",
    system_prompt="You are a research assistant.",
)

# The agent has built-in tools: write_todos, ls, read_file,
# write_file, edit_file, execute, and subagent spawning
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "Research AI agent frameworks and write a report"
    }]
})

Key insight: With Deep Agents, you don't define nodes, edges, or state schemas. The agent comes pre-wired with planning tools, a filesystem, subagent spawning, and context management. You just give it a task and a model.

06 Deep Agents Core Capabilities

What makes it more than just "LangGraph with a prompt".

Planning & Task Decomposition

Deep Agents includes a built-in write_todos tool. When given a complex task, the agent automatically breaks it into discrete steps, tracks progress, and adapts the plan as new information emerges. No graph definition needed — the agent manages its own workflow.

Virtual Filesystem

The agent gets ls, read_file, write_file, and edit_file tools out of the box. This is powerful for two reasons:

📂 Context Management

  • Offload large data to files instead of stuffing context
  • Read/write artifacts (scripts, notes, drafts)
  • Prevents context window overflow

🔌 Pluggable Backends

  • StateBackend — in-memory (ephemeral)
  • LocalBackend — real filesystem
  • SandboxBackend — isolated execution
  • Build your own custom backend

Subagent Spawning

Deep Agents — Subagent Delegation Python 
from deepagents import create_deep_agent

# Create a supervisor agent that can spawn subagents
agent = create_deep_agent(
    model="openai:gpt-4o",
    system_prompt="""You are a project manager.
    When tasks are complex, delegate them to subagents.
    Each subagent gets its own context and can work independently.""",
)

# The agent can spawn subagents automatically for:
# - Context isolation (each subagent has clean context)
# - Parallel work (multiple subagents at once)
# - Specialization (different prompts for different tasks)
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "Build a web scraper and a data analyzer for the scraped data"
    }]
})

When to use Deep Agents

  • Complex, multi-step tasks — research, coding, analysis that requires planning
  • Large context management — working with files, scripts, and artifacts
  • Long-running sessions — auto-summarization keeps the agent effective
  • Task delegation — breaking work across specialized subagents
  • Coding agents — file system access + shell execution + planning
  • Human-in-the-loop — built-in approval workflows for sensitive ops

When NOT to use Deep Agents

  • Simple chains — RAG, classification, summarization → use LangChain
  • Custom orchestration — you need fine-grained graph control → use LangGraph
  • Minimal overhead — Deep Agents adds its own tools and prompts
  • Deterministic workflows — if every step is known, a graph is clearer

07 Side-by-Side Comparison

The same concepts, three different levels of abstraction.

Dimension LangChain Framework LangGraph Runtime Deep Agents Harness
Core model Chain / pipeline Directed graph (DAG + cycles) Pre-built agent with tools
Control flow Implicit (LLM decides in agent loop) Explicit (you define transitions) Agent-managed (planning tools)
State management Conversation memory (messages) Typed state dict (arbitrary data) Files + conversation + auto-summarization
Cycles / loops Single agent loop only Any node can loop to any node Agent plans and re-plans autonomously
Branching Limited (if/else in code) Conditional edges with routing functions Agent decides based on todo list
Persistence Manual / via memory classes Built-in checkpointing (auto-save state) Filesystem + long-term memory
Human-in-the-loop Hack with callbacks Native interrupt() + resume Built-in interrupt_on parameter
Multi-agent Manual coordination Subgraphs + custom edges Built-in subagent spawning
Context management Conversation window only Manual via state reducers Auto-summarization + file offloading
Streaming Token-level from LLM Node-level + token-level Full agent streaming
Boilerplate Low — simple setup Medium — define state, nodes, edges Minimal — just create and run
Learning curve Lower — simpler mental model Higher — graph thinking required Medium — understand the tools

They're layers, not competitors. Deep Agents uses LangGraph for orchestration, which uses LangChain for model/tool abstractions. Each layer adds a different kind of power. Pick the level that matches your needs.

08 Same Task, Three Approaches

Build a "research and summarize" agent all three ways.

The task

Given a topic, search for information, summarize it, and if the summary is too short, search for more and try again. Output the final summary.

LangChain Approach

LangChain — Hope the LLM handles the loop Python 
# You put ALL the logic into the prompt
prompt = ChatPromptTemplate.from_template("""
Research the topic: {topic}

Instructions:
1. Search for information using the search tool
2. Write a summary (at least 200 words)
3. If your summary is under 200 words, search again
   with more specific queries and rewrite
4. Keep going until you have a comprehensive summary

Output ONLY the final summary.
""")

agent = create_tool_calling_agent(llm, [search_tool], prompt)
executor = AgentExecutor(agent=agent, tools=[search_tool], max_iterations=10)
result = executor.invoke({"topic": "AI safety"})

Problems: The LLM might not follow the loop correctly. You can't see how many iterations it did. You can't resume if it fails at iteration 8. You can't pause to approve intermediate results. The "200 words" check is in the LLM's head, not in code.

LangGraph Approach

LangGraph — Explicit control flow Python 
class State(TypedDict):
    topic: str
    summaries: Annotated[list, operator.add]
    final_summary: str
    word_count: int

def search_node(state):
    results = search(state["topic"])
    return {"summaries": [results]}

def summarize_node(state):
    combined = " ".join(state["summaries"])
    summary = llm.invoke(f"Summarize: {combined}").content
    return {"final_summary": summary, "word_count": len(summary.split())}

def is_sufficient(state):
    # Deterministic check — not in the LLM's head
    if state["word_count"] >= 200:
        return "end"
    if len(state["summaries"]) >= 5:
        return "end"  # safety limit
    return "search"  # loop back!

graph = StateGraph(State)
graph.add_node("search", search_node)
graph.add_node("summarize", summarize_node)
graph.add_edge(START, "search")
graph.add_edge("search", "summarize")
graph.add_conditional_edges("summarize", is_sufficient)
app = graph.compile()

The difference is control. The LangGraph version has a deterministic word-count check, a safety limit, visible iteration count, and the ability to checkpoint state at each step. The logic is in your code, not in the LLM's instructions.

Deep Agents Approach

Deep Agents — Let the agent plan and manage itself Python 
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="openai:gpt-4o",
    system_prompt="""Research the given topic thoroughly.
    Use the write_todos tool to plan your research steps.
    Save intermediate findings to files using write_file.
    The final summary should be at least 200 words.""",
)

# The agent will:
# 1. Create a todo list (plan)
# 2. Search and save results to files
# 3. Read files and synthesize a summary
# 4. Check its own word count and search more if needed
# 5. Write the final report to a file
result = agent.invoke({
    "messages": [{"role": "user", "content": "Research AI safety"}]
})

The tradeoff: Deep Agents requires the least code — no state schema, no node functions, no graph definition. The agent manages its own workflow using built-in planning tools. But you give up explicit control over every step. The agent decides how to decompose and execute the task. Good for exploratory work; less ideal when you need deterministic guarantees.

12 Common Pitfalls

Mistakes people make (and how to avoid them).

❌ Pitfall 1: "I'll use LangGraph for everything"

LangGraph adds complexity. If your task is a simple chain, using LangGraph means more code, more concepts, more debugging surface — for zero benefit. Start with LangChain. Graduate to LangGraph when you feel the pain.

❌ Pitfall 2: "I'll put all the logic in the prompt"

If your LangChain agent prompt has 5 paragraphs of conditional instructions ("if X then do Y, unless Z..."), you've outgrown the agent loop. Move the control flow to code with LangGraph. Prompts are for instructions, not for programming.

❌ Pitfall 3: "LangGraph is just for multi-agent"

LangGraph is great for multi-agent, but it's also great for single agents with complex workflows. A single research agent with retry logic, quality checks, and human approval is a perfect LangGraph use case — no second agent needed.

❌ Pitfall 4: "I need to pick one forever"

They're layers, not religions. LangGraph uses LangChain components inside its nodes. Deep Agents uses LangGraph under the hood. Start with LangChain chains, graduate to LangGraph when you need control, and use Deep Agents when you need autonomous, complex agents. The migration path is smooth.

❌ Pitfall 5: "Deep Agents is just a LangChain wrapper"

Deep Agents isn't just a prompt wrapper — it adds real infrastructure: planning tools, virtual filesystem, subagent orchestration, and auto-summarization. These are capabilities you'd have to build manually with LangGraph. But it's also not magic — the agent still uses an LLM to decide how to use these tools.

❌ Pitfall 6: "I'll use Deep Agents for simple tasks"

Deep Agents adds overhead — built-in tools, planning prompts, filesystem operations. For a simple "summarize this text" task, all that machinery is wasted. Use the right level: LangChain for simple, LangGraph for orchestrated, Deep Agents for autonomous.

✅ The golden rule

Use the simplest thing that works. Add complexity only when you have a specific problem that the simpler approach can't solve. You'll know when you hit that wall — your agent will be unreliable, un-debuggable, or un-extensible.

14 Cheat Sheet

Pin this. Reference it when starting a new project.

🟢 Use LangChain when:

  • Linear pipeline: input → process → output
  • Simple RAG (retrieve + generate)
  • Text classification / extraction / summarization
  • Single agent with straightforward tool use
  • You want minimal boilerplate and fast prototyping
  • Conversation memory is enough state

🔴 Use LangGraph when:

  • Cycles: retry loops, iterative refinement
  • Conditional branching: different paths based on results
  • Multi-agent systems: supervisor, debate, swarm
  • Human-in-the-loop: pause for approval mid-execution
  • State persistence: survive crashes, resume later
  • Complex state: accumulate data across many steps
  • You need visibility into exactly which step you're on
  • Long-running workflows (minutes to hours)

🟡 Use Deep Agents when:

  • Complex, multi-step tasks that need planning and decomposition
  • Large context management — working with files, scripts, artifacts
  • Long-running sessions — auto-summarization keeps agent effective
  • Coding agents — filesystem + shell + sandbox + planning
  • Autonomous subagent spawning for task delegation
  • You want batteries-included without building a graph

🔵 The mental model:

LangChain = "Here are the tools. Go figure it out." (LLM controls flow)

LangGraph = "Here's the blueprint. Follow it." (You control flow, LLM does the work)

Deep Agents = "Here's a capable assistant with its own desk, filing cabinet, and subordinates." (Pre-built agent system)

Start simple. Graduate up the stack when you feel the pain.

   YES → Deep Agents    NO → keep going

   YES → LangGraph    NO → keep going

   YES → LangGraph    NO → keep going

4. Is it a MULTI-AGENT system with explicit communication patterns?
   YES → LangGraph    NO → keep going

   YES → LangChain    You're done!
